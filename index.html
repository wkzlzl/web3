::two-step:: 사고방식

어떤 쪽이 원인이고 어떤 쪽이 결과인지 추론하는 것.
오른쪽이 사람이 하는 일이면서 원인, 그것에 대한 결과가 왼쪽이면서 기계가 하는 일이다.
원인을 부르는 여러가지 표현이있다. 코드,소스,약속이란 의미의 언어,
기계도 이해할 수 있고 사람도 이해할 수 있는, 컴퓨터 언어이다.

결과를 부르는 표현, 애플리케이션 응용프로그램, 앱 또는 프로그램,
다양한 표현들은 같은 대상을 바라보는 관점에 따라 달라지는 것에 불과하다.

코딩은 원인을 통해서 결과를 만든다는 것 코딩을 이해하는 핵심.

 Html의 장점 첫번째 쉽다. 두번째 중요하다. 웹 페이지를 만드는 기본적인 컴퓨터 언어, 쉬우면서 중요하다.

Web은 Public domain이다.

::three-step:: TAG

TAG란?

특정 요소의 시작과 끝을 표시하며 요소의 구현화를 위한 기능이다.
대략적으로 150여 가지의 태그가 있으며 속성(Attribute)을 사용하여 태그의 추가적인 기능이 사용 가능하다.

ex) <a> 는 시적으로 표현한 닻(anchor)의 약자이며 링크(link)의 기능을 가지고 있다.
배에서 닻을 내려 정박 하듯이 웹 페이지간의 연결을 주도한다. 추가적인 속성인 href(hypertext reference)을 사용해야 링크가 된다.
target = <_blank> 속성을 사용하면 새로운 탭이 열리고, title = "???" 속성은 링크에 대한 툴팁을 생성해낸다.

tip.1 자주 쓰이는 태그를 기반으로 순위를 표시한 사이트 "태그 랭킹 사이트"

tip.2 Html을 만든 사람들은 본문 내용을 <body>로 묶기로 약속되었고 <body>를 설명하는 태그로 <head>태그를 쓰기로 약속하였다.
그리고 이 두가지 고위급 태그는 최고위 태그인 <html> 안에 포함된다.
관용적으로 <html> 위로는 <!doctype html> 표시하기로 되어있다.

HTML

Hypertext Markup Language(HTML) is the standard markup language for creating web pages and web applications.
Web browsers receive HTML documents from a web server or from local storage and render them into multimedia web pages.
HTML describes the structure of a web page semantically and originally included cues for the appearance of the document.

::five-step:: 인터넷과 웹

Internet VS Web 같다? 다르다?

Internet : 도시,도로,운영체제 등 상위의 존재

Web : 도시에 세워진 빌딩, 도로를 달리는 자동차, 운영체제에 속한 웹,
하위의 존재(WEB,FTP,E-mail 등등 수많은 서비스들이 있음)

Internet 과 Web 의 역사

1960년 - 1990 년
  • 1960년 인터넷(internet)등장
  • 핵 전쟁 후의 시대, 화두가 된 "핵"
  • 미국은 핵 공격 피해를 입게 된 상황을 시뮬레이션으로 돌려보게 되고 통신 시스템의 취약점을 발견
  • 중앙집중적이었던 통신 시스템은 파괴 당할 시 미국의 통신 마비를 일으키는 결과를 초래함
  • 핵 공격에도 견딜 수 있는 강인한 통신 시스템이 대두됨. 그로인해 탄생한 인터넷(internet)

인터넷(internet)의 구조
  • WEB의 탄생지, 스위스 제네바의 '유럽입자물리연구소(CERN)'127개국에서 12000명의 과학자와 기술자들 모인 곳
  • '피터 힉스'라는 박사는 아주 작은 입자를 고속으로 회전시켜서 입자가 충돌을 일으켰을 때 커다란 에너지가 발생할 것이라 가설을 세움
  • 그것을 연구하기 위한 '강입자 가속기(LHC)'를 개발함. 크기 27km의 둘레의 크기이며 인류가 만든 가장 거대한 발명품
  • 가설로 진행된 실험은 최근 성공하여 '힉스' 물질을 발견함(노벨물리학상 수상)
  • 1980년 '팀 버너스리'라는 프로그래머 입사
  • '팀 버너스리'는 Enquire 프로그램 개발, 웹의 전신중 하나, 현재는 소실 됨
  • 1990년. '팀 버너스리' 10년 동안 도전했던 과제와 인터넷을 결합. WEB의 시작
  • 1990.10월 세계 최초로 웹페이지를 만드는 편집기 개발
  • 1990.11월 세계 최초의 웹브라우저인 World Wide Web을 제작
  • 1990.12월 웹서버 프로그램 개발. info.cern.ch 주소를 부여

::six-step:: 서버와 클라이언트

  • 인터넷이 동작하기 위해선 두대의 컴퓨터가 필요하다.
  • 각각의 컴퓨터엔 웹 브라우저, 웹 서버를 깔아 두고 인터넷으로 연결
  • 웹 서버 컴퓨터엔 index.html이란 파일이 있고 http:/info.cern.ch 라는 주소를 가지고 있는 상황
  • 웹 브라우저 컴퓨터는 인터넷의 전기 신호를 통해 서버 컴퓨터 안에 있는 index파일을 요구할 수 있고 서버 컴퓨터가 응답한다면 웹 브라우저 컴퓨터에 출력됨
  • 두 컴퓨터의 관계를 자본주의 사회의 시대에서 자주 쓰이는 고객과,사업자로 명칭함.
  • 웹 브라우저 = 고객 = 클라이언트(client)
  • 웹 서버 = 서비스 제공자 = 서버(server)
